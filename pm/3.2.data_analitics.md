## Вывод по данным для MVP

Для запуска MVP за **3 месяца** достаточно использовать в основном данные, помеченные как **MVP** (регион, тип доставки, локация склада, исторические сроки, сезонность и т.д.).  

Это позволяет построить базовый прогноз диапазона доставки и вероятности без сложных внешних интеграций.  

Дополнительные данные (загруженность складов, реал-тайм отклонения, погода и глубокая персонализация) будут подключаться на следующих итерациях для повышения точности модели.

## Ключевые метрики аналитики функции

- Точность прогноза (±1 день)
- MAE (средняя ошибка прогноза)
- % заказов с корректным ETA
- Снижение обращений в поддержку по доставке
- Рост CR оформления заказа
- Retention 30/90 дней среди пользователей, видевших прогноз
- % пользователей, взаимодействовавших с прогнозом доставки
