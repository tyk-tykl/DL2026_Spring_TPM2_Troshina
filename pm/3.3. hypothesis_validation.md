# 3.3. Проверка гипотезы

## Методы проверки гипотезы Smart Delivery Prediction

| Метод                    | Суть метода                              | Как применяется                              | Что измеряем                     | Ожидаемый результат                  |
|--------------------------|------------------------------------------|----------------------------------------------|----------------------------------|--------------------------------------|
| Fake Door Test           | Добавление интерфейса без реализации    | Кнопка «Точная дата доставки» в карточке    | CTR, количество кликов           | Высокий интерес пользователей        |
| Прототипирование (Figma) | Интерактивный макет                      | Прототип экрана с прогнозом доставки         | Usability, поведение пользователей | Легкость использования функции       |
| Анализ текущих данных    | Изучение существующих проблем            | Анализ жалоб, возвратов, drop-off            | Частота отмен и негативных отзывов | Подтверждение наличия боли           |
| Консьерж-тест            | Ручная имитация функции                  | Ручной расчёт даты доставки для тестовых пользователей | Влияние на конверсию и доверие   | Рост вероятности покупки             |

**Рекомендованный подход:**  
На первом этапе — **Fake Door Test** (минимальные затраты, быстрый результат).  
Для более глубокой валидации — комбинация анализа текущих данных + Консьерж-тест.

**Вывод:**  
Комбинация методов позволит сначала быстро проверить интерес аудитории, а затем подтвердить реальную ценность функции для бизнеса и пользователей.
